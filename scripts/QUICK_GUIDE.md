# 클로드 코드 ↔ 옵시디언 연동 가이드

## [목표] 핵심: 어디서든, 언제든 문서화 요청만 하면 자동 저장!

## 사용 예시

### 1. 개발하다 막힌 부분 해결 후
```
나: "Docker 네트워킹 문제 해결했는데 이거 옵시디언에 정리해줘"
클로드: [자동으로 02_Learning/Docker/에 저장]
```

### 2. 프로젝트 작업 완료 후
```
나: "my-project API 개발 완료했으니 문서화해줘"
클로드: [자동으로 01_Work/Projects/my-project/에 저장]
```

### 3. 새로운 작업 추가
```
나: "멀티모듈 템플릿 구성 TODO에 추가"
클로드: [자동으로 00_HOME/daily/2026/03/1주차.md에 추가]
```

## 자동 분류 시스템

문서 내용을 분석해서 자동으로 카테고리 분류:
- Spring 관련 → `02_Learning/Spring/`
- React 관련 → `02_Learning/React/`
- Docker/인프라 → `02_Learning/Docker/`
- 알고리즘 → `02_Learning/Algorithm/`
- 기타 → `02_Learning/ETC/`

## 저장되는 내용

### 학습 노트
- 문제 상황
- 해결 방법
- 코드 예제
- 핵심 학습 포인트
- 참고 자료

### 프로젝트 문서
- 작업 날짜
- 작업 유형
- 상세 내용
- 관련 코드

### 주간 작업
- TODO/DONE 상태
- 프로젝트별 분류
- 작업 날짜

## 마법의 명령어들

```
"이거 옵시디언에 정리해줘"
"문서화해줘"
"노트에 저장해줘"
"학습 내용 정리해줘"
"작업 목록에 추가해줘"
```

## 파일 위치

모든 파일은 아래 위치에 자동 저장:
```
$HOME/Documents/Obsidian Vault/
├── 00_HOME/daily/2026/03/  (주간 작업)
├── 01_Work/Projects/        (프로젝트 문서)
└── 02_Learning/              (학습 노트)
    ├── Spring/
    ├── React/
    ├── Docker/
    ├── Algorithm/
    └── Claude_Sessions/      (클로드 세션)
```

---
[팁] **팁**: 클로드 코드가 알아서 내용을 분석해서 적절한 위치에 저장하니, 그냥 "정리해줘"라고만 해도 됩니다!
